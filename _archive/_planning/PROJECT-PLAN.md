Ship a public repo and landing page for an AI-powered SME knowledge-transfer interview, with a fully functional stub experience and a clear path to live mode.

1. Product and scope

   - Keep the stub experience fully usable without keys

2. UX / visual system
   - Match tPP landing-page pattern exactly
   - Produce favicon, hero image, and social preview asset

3. Build
   - Create the web app interview flow
   - Implement artifact generation for the four outputs
   - Add settings/config hooks for live mode keys

4. Documentation
   - Maintain README, decisions log, and setup instructions
   - Include a complete example walkthrough

## Delivery Order
1. Read and align on brief
2. Lock scope and repo structure
3. Build stub experience end-to-end
4. Add assets and landing page polish
5. Document setup, example, and handoff

## Risks
- Live-mode integrations may add complexity, so stub mode stays the release baseline
- Asset quality needs to match sibling products closely
- Repo structure must stay simple enough for a non-technical user to run locally
